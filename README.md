## The Problem

English-speaking practice in Bangladesh is either **absent** (grammar-only coaching, no speaking feedback), **misfit** (global apps built for Western learners, blind to Bangla phonology), or **expensive** (private tutors: BDT 8,000–24,000/month; IELTS exam fee alone: BDT 28,450). Bangla phonology also lacks several English phonemes (`/v/`, `/æ/`, `/θ/`, `/ð/`, `/z/`, `/ɪ/`, `/w/`), producing systematic mispronunciations that generic apps never detect, let alone explain.

## The Solution

**Articulate AI** is a full spoken-English learning ecosystem purpose-built for Bengali speakers:

- **Phoneme-level pronunciation scoring** via Azure Cognitive Services, with Bangla-language coaching tips targeting the exact sounds a learner struggles with
- **CEFR-structured curriculum** (A1–C1) built on the Oxford 3000 word list — 8 chapters, 40 lessons, 7,640 words
- **IELTS speaking simulation** with a turn-by-turn AI voice examiner and a structured band report (4–9)
- **Pronunciation-driven spaced repetition** — an SM-2 flashcard scheduler where speaking accuracy (not just recall) determines review intervals
- **Gamification** — XP, streaks, levels, achievement badges, and a leaderboard to keep learners coming back

All at a fraction of the cost of a human tutor.

## 2. AI Innovation

**1. Denoise → Score pipeline.** Recordings pass through a dedicated **DeepFilterNet3** neural denoiser (Python/FastAPI microservice) before Azure scoring — handling non-stationary noise (traffic, babble) that simple noise gates miss, with peak-normalization to prevent volume drop. Falls back gracefully to raw audio if the service is unreachable.

**2. Two-pass free-speech assessment.** For IELTS conversation turns, where no reference text exists in advance, Azure Speech-to-Text first transcribes the utterance — then that transcript is fed back into the Pronunciation Assessment API as the reference, enabling scored feedback on completely spontaneous speech.

**3. Bangla-risk phoneme targeting.** A curated set of 7 IPA phonemes that Bengali speakers systematically mispronounce is tracked per user, with native-language (Bangla) articulation guidance generated by GPT-4o-mini — not generic pronunciation tips.

**4. Score-driven SRS.** The standard SM-2 spaced-repetition algorithm normally uses subjective recall confidence (0–5) as its quality input. Here, the actual pronunciation accuracy score (0–100%) drives the quality parameter — so words you *can't say correctly* resurface sooner than words you can.

**5. LLM-generated curriculum content.** The full Oxford 3000 dataset was parsed from its native 4-column PDF layout via a custom parser, then enriched with 160 GPT-4o-mini–generated lesson phrases (using each lesson's exact vocabulary for contextual reinforcement) and pre-generated Azure Neural TTS audio for every word and phrase.

## 3. Architecture

```
┌─────────────────────┐      ┌──────────────────────────-┐      ┌─────────────────────────────┐
│      Frontend       │      │         Backend           │      │      AI & Cloud Services    │
│  React 19 + Vite 8  │      │   Node.js + Express 5     │      │                             │
│  Tailwind CSS v4    │      │                           │      │  Azure Cognitive Services   │
│  React Router v6    │◄────►│  PostgreSQL (Neon)        │◄────►│   • Pronunciation Assessment│
│  Recharts           │ JWT  │  JWT auth (access+refresh)│ REST │   • Speech-to-Text          │
│  Socket.io-client   │      │  Socket.io + LISTEN/NOTIFY│  +   │   • Neural Text-to-Speech   │
│  MediaRecorder API  │      │  SM-2 SRS engine          │Socket│                             │
│  localStorage cache │      │  Prometheus + Winston     │      │  OpenAI GPT-4o-mini         │
└─────────────────────┘      │  Swagger API docs         │      │  DeepFilterNet3 (FastAPI)   │
                             └──────────────────────────-┘      │  Cloudinary · Mailtrap      │
                                                                └─────────────────────────────┘
```

**Real-time layer:** Socket.io, authenticated at handshake via JWT, paired with PostgreSQL `LISTEN/NOTIFY` — a DB trigger on notification inserts pushes directly to the right user's socket without extra application-layer plumbing.

## 4. Tech Stack

**Frontend** — React 19 · Vite 8 · Tailwind CSS v4 · React Router v6 · Recharts · Socket.io-client · Lucide Icons

## 6. Getting Started

```bash
# Backend
cd backend
cp src/.env.example .env   # fill in DB, JWT, Azure, OpenAI, Cloudinary credentials
npm install
npm run dev

# Frontend
cd frontend
npm install
npm run dev

# Denoiser microservice (optional — falls back gracefully if not running)
cd denoiser-worker
python bootstrap.js  # sets up venv + dependencies on first run
```

Backend API docs available at `/api/docs` once running.
